# DeepDocs を試す手順

## 前提

- GitHubアカウントがあること
- GitHubにリポジトリを作成できること

## ステップ1: GitHubにリポジトリを作成してpush

```bash
# このフォルダに移動
cd deepdocs-trial

# Gitリポジトリを初期化
git init
git add .
git commit -m "初期コミット: タスク管理APIのコードと設計書"

# GitHubに空のリポジトリを作成（ブラウザで作るか、gh CLIで作る）
gh repo create deepdocs-trial --public --source=. --push
# ※ プライベートリポジトリでもFreeプランで使える
```

## ステップ2: DeepDocsをインストール

1. https://deepdocs.dev/ にアクセスし、GitHubアカウントでサインイン
2. GitHub Appのインストール画面で、作成したリポジトリ（deepdocs-trial）を選択してインストール
3. DeepDocsのダッシュボードで該当リポジトリを開く
4. Repository Settings で以下を設定:
   - **Branch**: `main`
   - **Source Code Context**: `src/` フォルダを指定（コード変更の監視対象）
   - **Target Docs**: `docs/` フォルダを指定（更新対象のドキュメント）
5. 「Start Scan」をクリックして初回スキャンを実行

※ 参考: [DeepDocs Quickstart（公式）](https://docs.deepdocs.dev/get-started/quickstart/)

## ステップ3: コードを変更してPRを作成

設計書が自動更新されるか試すために、コードに変更を加える。

### 変更例: 「優先度」フィールドを追加する

```bash
# 新しいブランチを作成
git checkout -b feature/add-priority
```

以下の変更を行う:

**Task.java に `priority` フィールドを追加:**
```java
private Priority priority;  // 追加

public Priority getPriority() { return priority; }  // 追加
public void setPriority(Priority priority) {         // 追加
    this.priority = priority;
    this.updatedAt = LocalDateTime.now();
}
```

**Priority.java を新規作成:**
```java
public enum Priority {
    LOW,
    MEDIUM,
    HIGH
}
```

**TaskService.java の createTask を修正:**
```java
public Task createTask(String title, String description, Priority priority) {
    Task task = new Task();
    task.setTitle(title);
    task.setDescription(description);
    task.setStatus(TaskStatus.TODO);
    task.setPriority(priority);  // 追加
    return taskRepository.save(task);
}
```

**TaskController.java の createTask を修正:**
```java
@PostMapping
public ResponseEntity<Task> createTask(@RequestBody CreateTaskRequest request) {
    Task task = taskService.createTask(
        request.getTitle(),
        request.getDescription(),
        request.getPriority()  // 追加
    );
    return ResponseEntity.status(HttpStatus.CREATED).body(task);
}
```

### コミットしてPRを作成

```bash
git add .
git commit -m "feat: タスクに優先度(priority)を追加"
git push origin feature/add-priority

# PRを作成
gh pr create --title "feat: タスクに優先度を追加" --body "タスクにLOW/MEDIUM/HIGHの優先度フィールドを追加"
```

## ステップ4: DeepDocsの動作を確認

- コミットがpushされると、DeepDocsが自動でコード差分をスキャンする
- `docs/design.md` が古くなっていると判定されると、`deepdocs-update/` ブランチに更新内容をコミットし、PRを自動作成する
- PRの内容を確認して、設計書が正しく更新されているかレビューする

### 確認ポイント

- [ ] `docs/design.md` のデータモデルに `priority` フィールドが追加されているか
- [ ] `Priority` のEnum定義が追記されているか
- [ ] API仕様の `POST /api/tasks` のリクエストに `priority` が追加されているか
- [ ] クラス構成に `Priority` が追加されているか
